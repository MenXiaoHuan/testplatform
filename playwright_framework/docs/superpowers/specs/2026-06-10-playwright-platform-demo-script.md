# Playwright Platform Demo Script

## Demo Goal

用 5 分钟向观众展示平台已经具备的最小 MVP 能力：

1. 管理测试仓库
2. 管理执行场景
3. 触发任务执行
4. 查看任务详情、报告和附件入口

## Demo Preconditions

演示前确保以下服务已经启动：

1. 后端：`http://localhost:8080`
2. 前端：`http://localhost:4173`

如需重新启动，参考：

- [Runbook](file:///Users/bytedance/test_platform/playwright_framework/docs/superpowers/specs/2026-06-10-playwright-platform-runbook.md)

## Demo Flow

### 1. 打开平台首页

操作：

1. 浏览器打开 `http://localhost:4173/`
2. 展示左侧导航与三个主菜单：
   - 测试仓库
   - 执行场景
   - 任务列表

讲解要点：

1. 这是独立于测试仓库之外的平台管理台
2. 平台不负责写测试代码，只负责接入、调度和查看结果

### 2. 展示仓库管理页

操作：

1. 进入 `/repos`
2. 点击“新增仓库”
3. 展示可填写的字段：
   - 仓库名称
   - Git 地址
   - 默认分支
   - 安装命令
   - 执行命令模板
   - 报告目录

讲解要点：

1. 平台通过约定的接入契约管理多个 Playwright 仓库
2. 一个仓库可以对应多个执行场景

### 3. 展示场景管理页

操作：

1. 进入 `/scenes`
2. 点击“新增场景”
3. 展示场景配置项：
   - 所属仓库
   - 分支
   - 选择方式
   - 筛选值
   - 执行命令

讲解要点：

1. 场景是可复用的执行配置，不是一次运行结果
2. 场景可以抽象出“跑哪个分支、哪些用例、用什么命令”

### 4. 展示任务触发入口

操作：

1. 在场景列表中点击“执行”
2. 跳转或切换到任务列表页 `/tasks`
3. 展示任务记录中的字段：
   - 任务 ID
   - 分支
   - 状态
   - 触发方式
   - 耗时

讲解要点：

1. 每次点击执行都会生成一条任务记录
2. 后续会继续增强状态阶段和结果摘要

### 5. 展示任务详情页

操作：

1. 在任务列表点击“详情”
2. 打开 `/tasks/:id`
3. 展示任务详情中的内容：
   - 基础信息
   - 报告地址
   - 日志地址
   - 附件列表

讲解要点：

1. 任务详情是报告与附件的聚合入口
2. 平台的目标不是替代 Allure，而是给 Allure 和附件提供统一入口
